# Template Usage Memory

- This repository is a starter template. Keep changes product-neutral unless the
  new repository has already chosen a product direction.
- New product code should live in `apps/app/src/features/<feature>` with routes
  under `apps/app/src/routes/_app`.
- Shared UI changes belong in `packages/ui` only when they are reusable across
  apps.
- Keep Cloudflare resource IDs and secrets as placeholders or environment
  values.
- Keep setup, deployment, and future plans documented under `docs/`.
- Do not add CI/CD by default. The baseline workflow is local validation plus
  manual deploy.
